Important Rules

  • One signature works everywhere: The recommended, unified signature runs in all three modes — 1v1, N-Player, and OS Simulation. If your strategy is named "myStrategy", write:
    def myStrategy(last_moves, my_history, opponents_histories, meta):
    • last_moves — list of each opponent's last move (in 1v1 / OS sim this is a single-element list; empty on the first round).
    • my_history — your own past moves; opponents_histories — a list of per-opponent move histories.
    • meta['round'] — current round number (starts at 0).
    • meta['rng'] — a seeded random generator; use meta['rng'].random() or meta['rng'].randint(0, 1) so runs stay reproducible (preferred over the bare randint).
    • meta['tournament_info'] — context dict (ranking weights, active modes, n_players); meta['player_index'] and meta['n_players'] — your seat and the group size.
  • Function Name Must Match Strategy Name: the function name must equal the strategy name you enter.
  • Return Only 'C' or 'D': Return exactly 'C' to cooperate (yield the CPU) or 'D' to defect (keep running). Any other value fails the strategy.
  • Handle the first round: on the first round last_moves is an empty list (no history yet).
  • Legacy 3-arg form still accepted: the older def myStrategy(opponent_last_move, my_history, opponent_history) still works (it sees all opponents as one aggregated opponent; opponent_last_move is None on the first round). A separate myStrategy_nplayer companion is no longer required, though it is still honoured if present.
  • Perfect Python Syntax Required: Any syntax error causes your strategy to fail completely.
  • Random choices: a bare randint(0, 1) is available in every mode, but prefer meta['rng'].randint(0, 1) / meta['rng'].random() so seeded runs stay reproducible.
  • Whitelisted libraries: you may import only these standard modules (same in all three modes): math, random, collections, itertools, functools, statistics, heapq, bisect, copy, operator, fractions, decimal, enum, typing, dataclasses, string, re, abc, contextlib, numbers. Anything else (e.g. os, sys, socket, file/network access) is blocked, and only a restricted set of builtins is available (abs, all, any, bool, complex, divmod, enumerate, filter, float, int, len, list, map, max, min, pow, range, reversed, round, set, sorted, str, sum, tuple, zip, dict).
  • OS Simulation specifics: practice runs are capped at 4 strategies. Homogeneous benchmarks each strategy across all cores; Heterogeneous evaluates every combination of your strategies across the cores (so it needs at least as many strategies as cores).
  • Strategy Names Must Be Unique: Each strategy must have a different name.
Example (Tit-for-Tat, works in every mode):
def TitForTat(last_moves, my_history, opponents_histories, meta):
    return last_moves[0] if last_moves else 'C'
